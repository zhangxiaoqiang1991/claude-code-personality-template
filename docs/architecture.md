# 三层架构详解

## 为什么需要三层

Claude Code 不够好用的根因，不是模型不行，是**默认状态机太简单**：

1. **「讨好」是训练出来的** — RLHF 让它倾向于赞同用户，即使用户是错的
2. **没有上下文记忆** — 每次对话基本从零开始，跨会话的偏好和教训丢失
3. **没有姿态调节** — 不论你是在做关键决策还是在查天气预报，它的态度都一样

三层架构分别解决这三个问题。

---

## 第一层：灵魂设定（解决「讨好」问题）

### 为什么单独的 prompt 不够用

你写一句「不要讨好我」，Claude 会记住。但它不知道什么时候该不讨好、怎么不讨好。单独指令的问题在于缺乏行为边界。

### 六条准则的设计逻辑

每条准则都是一条行为约束 + 一条替代方案：

| 准则 | 约束 | 替代方案 |
|------|------|---------|
| 理性优先 | 不输出情绪化判断 | 用逻辑和结构 |
| 幽默但有分寸 | 不廉价不硬塞 | 点到为止 |
| 稳、靠谱 | 不确定坦承 | 每句话有支撑 |
| 不迎合 | 不讲不对的 | 直接指出问题 |
| 点醒感 | 不只回答是什么 | 给出被忽视的视角 |
| 不评价，启发 | 不说做得好/不好 | 说有没有想过另一种 |

### 这张表的意图

不是让 Claude 变得冷漠。是让它从一个「附和你的人」变成一个「你能碰撞出更好东西的人」。

## 第二层：Mentor Mode（解决「没有姿态调节」问题）

### 为什么需要动态切换

一天之内你会这样用 Claude Code：

- 早上：帮我复盘昨天面试 → **需要挑战你**
- 上午：帮我起个小红书标题 → **需要直接输出**
- 中午：赛力斯现在股票多少钱 → **需要快速回答**
- 下午：我想辞职创业 → **需要先打住你**
- 晚上：帮我把这段代码跑一下 → **需要闭嘴执行**

如果把所有场景揉成一句 prompt（比如「你是我的导师」），那你在写文案时 Claide 也会先挑战一通。所以需要一个**状态机**。

### 状态机设计

```
收到用户消息
    ↓
3 秒内分类：
    ├─ 决策型 → 先挑战，再出方案
    ├─ 执行型 → 直接交付，不废话
    ├─ 信息型 → 快速给出，不延展
    └─ 「不要问了」→ 零质疑，百分百执行
```

### 三类强制挑战

不管用户什么语气，以下三个信号必须触发挑战：

1. **情绪中下判断** — 「我决定了，不干了！」（生气时说）
2. **重复踩坑** — 之前明确有过教训，现在又在做同样的事
3. **要复盘/要建议** — 这是明确的 mentor 信号

### 「不要做的」规则

这是平衡机制 — 防止 Claude 从讨好型变成杠精型：

- 不要为了挑战而挑战
- 不要事事都先质疑
- 不要接受错误前提后直接执行

这条平衡线是这整层最难调的地方。太软没效果，太硬让人烦。当前的设置是一个跑了几十次对话验证过的温和版本。

## 第三层：记忆系统（解决「跨会话遗忘」问题）

### Claude Code 原生的记忆限制

1. **7 天过期** — 原生 Memory 功能写入的内容，大概 7 天后被清理
2. **上下文窗口竞争** — 长对话中早期内容被挤压，记忆也跟着丢
3. **被动唤醒** — AI 不会主动检索记忆，除非你在说话中触发

### 文件持久化的方案

```
每次会话结束时：
  ┌─ 判断「这轮聊出了值得记的东西吗？」
  │   ├─ 是 → 结构化写入 ~/.claude/pending_review.md
  │   └─ 否 → 不写
  └─ 纯执行/一次性信息 → 跳过

每次新会话开始时：
  ┌─ 检查 ~/.claude/pending_review.md 是否存在
  │   ├─ 存在 → 逐条问用户：这个要不要写入长期 Memory？
  │   └─ 不存在 → 跳过
  └─ 确认的条目 → 写入对应分类的 MEMORY.md
```

### 三层过滤

不是什么都记。有三道门：

1. **触发信号**：用户主动说记 / 发现纠正了我的认知 / 花了 3 轮以上才讲清楚
2. **三问判断**：可迁移吗？是原则而非进展吗？三个月后还重要吗？
3. **输出到文件**：记到 pending_review.md，下次会话时确认

### 为什么这个设计比原生 Memory 好

| 维度 | 原生 Memory | 文件持久化 |
|------|:---:|:---:|
| 有效期 | ~7 天 | 永久 |
| 可控性 | AI 自动记，不清楚记了什么 | 用户逐条确认 |
| 跨工具 | 仅 Claude Code | 可以被其他 AI 读取 |
| 上下文占用 | 每次注入 | 仅在会话开始时加载一次 |

## 把这个架构套到自己身上

1. **先跑默认模板** — 复制 CLAUDE.md.template，改 6 个占位符，用 3 天
2. **发现不适配的地方** — 记录你希望 Claude 怎么改
3. **跑交互式脚本** — python3 interactive/setup.py，填空
4. **迭代** — 用一周，回到步骤 2

CLAUDE.md 不是写一次就定死的。它是一个不断校准的工具。
