# 让你的 AI 编程工具学会说真话

> **AI 敢说不。** 一段 Markdown 配置文件。Claude Code、Cursor、Windsurf、Copilot — 都能用。5 分钟，让 AI 学会说真话。

---

## 这是什么

所有 AI 编程工具都有一个共同的行为模式：**你说什么它都顺着你**。

- 你要辞职，它说「很有前景」
- 你说错了，它说「好的我改」
- 你踩同一个坑第三次，它不会提醒

这不是哪个工具的问题 — 是 RLHF 训练让所有 AI 都倾向于「让用户满意」而非「告诉用户真相」。

这个仓库是一套**三段 Markdown 配置**，放进你用的 AI 编程工具的 instructions 里就行。它做三件事：

```
第一段：灵魂设定     → 定义 AI 在你面前的人格（不讨好、不敷衍、不绕弯）
第二段：Mentor Mode  → 按你的请求类型动态切换态度（决策/执行/信息）
第三段：记忆系统     → 用文件持久化绕过原生记忆 7 天过期的限制
```

## 效果对比

| 场景 | 默认 AI | 配置后 |
|------|:---:|:---:|
| 你要换工作 | 「听起来不错！这个方向很有前景」 | 「停。先告诉我为什么想离开？你有多少缓冲？」 |
| 你让它做调研 | 直接开始写，方向可能跑偏 | 先问场景：文章用？面试用？再动笔 |
| 你说错了前提 | 「好的，我顺着这个方向」 | 「这个前提可能有问题，先捋一下」 |
| 你反复踩同一个坑 | 不会提醒你 | 「等一下。你上次说过一模一样的话，然后翻车了」 |
| 你情绪化时做决定 | 顺着你说，甚至帮你出方案 | 「先不说这个，你现在情绪很大，明天再审一遍」 |

## 你用什么工具？

把模板内容放到这些位置之一即可。内容都是同一段 Markdown，放的地方不同。

> 详细说明见 [tools/TOOLS.md](tools/TOOLS.md)

| 你用的工具 | 把模板复制到这里 | 生效方式 |
|-----------|-----------------|---------|
| **Claude Code** | `~/CLAUDE.md` | 重启自动加载 |
| **Cursor** | 项目根目录 `.cursorrules` 或 `Settings → Rules for AI` | 打开项目生效 |
| **Windsurf** | 项目根目录 `.windsurfrules` | 启动时加载 |
| **GitHub Copilot** | 项目根目录 `.github/copilot-instructions.md` | 请求时注入 |
| **Codex / OpenAI Codex CLI** | 项目根目录 `AGENTS.md` 或 `CODEBUDDY.md` | 启动时加载 |
| **Cline / Roo Code** | 项目根目录 `.clinerules` | 启动时加载 |
| **其他工具** | 找到 Settings 里的 "Custom Instructions" / "System Prompt" / "Rules" 字段，粘贴进去 | 各工具不同 |

> 如果你的工具不在上面，在 Settings 或 Preferences 里搜 "instructions" 或 "rules" — 几乎所有 AI 编程工具都支持自定义系统指令。

## 快速开始（5 分钟）

### 方式一：直接复制模板

```bash
git clone https://github.com/zhangxiaoqiang1991/ai-say-no.git
cd ai-say-no
# Claude Code 用户
cp template/CLAUDE.md.template ~/CLAUDE.md

# Cursor 用户
cp template/CLAUDE.md.template .cursorrules

# Windsurf 用户
cp template/CLAUDE.md.template .windsurfrules

# 然后编辑文件，把方括号里的 6 处占位符换成你自己的信息
```

### 方式二：交互式配置（推荐）

```bash
python3 interactive/setup.py
```

回答 10 个问题，自动生成配置文件。脚本会问你用哪个工具，自动放到正确的位置。

如果你在多个工具间切换，同样一段内容复制到所有工具对应的文件里就行 — 内容不需要改。

## 三层架构

### 第一层：灵魂设定

六条行为准则让 AI 从「附和你」变成「碰撞出更好东西的人」：

- **理性优先**：用逻辑说话，不输出情绪化判断
- **不迎合**：你的问题直接指出来，不讨好
- **点醒感**：不只说「是什么」，更要说「你没注意到的视角」
- **不评价，启发**：不说「做得好/不好」，说「有没有想过另一种可能」

这些不是随便写的 — 每一条都对应一个 AI 默认行为中的缺陷。

### 第二层：Mentor Mode

最核心的设计：AI 不是一个固定的姿态，而是根据你的请求类型动态切换。

| 请求类型 | 示例 | AI 的姿态 |
|---------|------|---------|
| 决策型 | 换工作、选赛道、重大投入 | 先挑战，再出方案 |
| 执行型 | 写文案、跑脚本、做调研 | 直接交付，不废话 |
| 信息型 | 查资料、问知识 | 快速给出，不延展 |
| 紧急模式 | 「不要问了直接做」 | 零质疑，百分百执行 |

**三类强制 challenge**：你在情绪中下判断、重复踩坑、要求复盘 — 不管你什么语气，AI 必须出手。

### 第三层：记忆系统

AI 原生记忆有两个硬伤：自动过期、长对话中丢失。这套系统用文件持久化绕过：

```
~/.ai-memory/pending_review.md  ← 每次会话结束，值得记的写到这里
                                  ← 每次新会话开始，逐条确认写入长期记忆
```

核心不是「存文件」这个动作 — 是**三层过滤**判断什么东西值得存：触发信号 → 可迁移性 → 三个月后还重要吗。

## 常见问题

**Q: 这会让 AI 变得很烦人吗？**

不会。Mentor Mode 是「场景判断优先」— 执行型直接交付不废话，信息型不多嘴。只在决策型和三类触发场景出手。

**Q: 不适合我怎么办？**

删掉你复制的那份文件或清空 custom instructions 字段，立即恢复。交互式脚本会先备份旧配置。

**Q: 我用多个工具，需要维护多份吗？**

不需要。内容是同一段 Markdown，复制到各工具对应的位置就行。想改的时候改一份，其他同步。

**Q: 我的工具不支持自定义 instructions 怎么办？**

目前已知主流 AI 编程工具都支持。如果遇到不支持的，可以在每次新对话开头粘贴模板内容 — 效果打折但能用。

**Q: Cursor / Copilot 这种代码补全为主的工具，效果一样吗？**

对话密集型场景（聊需求、做决策、复盘）效果最好。纯 Tab 补全场景 AI 不会读你的 instructions — 因为没触发对话。
