# Claude Code 人格配置

> 5 分钟，让 Claude Code 从「你说什么都对的乙方」变成「敢说真话的战友」。

---

## 这是什么

Claude Code 默认是个老好人 — 你说什么它都附和，从不质疑你，做错了也只会说「好的我改」。

这不是 bug。是 RLHF 训练的结果。

但当你需要的是一个**能叫醒你的人** — 在你方向跑偏时纠正你、在你情绪化时按住你、在你重复踩坑时提醒你 — 默认的 Claude Code 做不到。

这个仓库给了你一套**三层架构**的 CLAUDE.md 配置方案：

```
第一层：灵魂设定     → 定义 Claude 在你面前的人格
第二层：Mentor Mode  → 按请求类型自动切换姿态（决策/执行/信息）
第三层：记忆系统     → 绕过 7 天过期，让 Claude 真正记住你
```

---

## 效果对比

| 场景 | 默认 Claude Code | 配置后 |
|------|:---:|:---:|
| 你要换工作 | 「听起来不错！这个方向很有前景」 | 「停。先告诉我为什么离开现在的？之前那份工作你说过同样的话。」 |
| 你让他做调研 | 直接开始，方向可能跑偏 | 先判断这是决策型还是执行型请求，再决定姿态 |

| 你反复踩同一个坑 | 不会提醒你 | 「等一下。上次你说过一模一样的话，然后翻车了。要不要先复盘？」 |
| 你情绪化时做决定 | 顺着你说 | 「你现在情绪比较大，这个决定要不要明天再审一遍？」 |

```bash
# 1. 克隆仓库
git clone https://github.com/zhangxiaoqiang1991/claude-code-personality-template.git

# 2. 复制模板
cp template/CLAUDE.md.template ~/CLAUDE.md

# 3. 编辑 ~/CLAUDE.md，把方括号里的内容换成你自己的
#    总共 6 处需要改

# 4. 重新打开 Claude Code，生效
```

## 交互式配置（推荐，10 分钟）

如果你不想手动改模板，运行交互式向导：

```bash
cd claude-code-personality-template
python3 interactive/setup.py
```

脚本会问你 10 个关键问题：

1. **怎么称呼你？** — 决定 Claude 对你的称呼
2. **你的职业背景** — 让 Claude 理解你的语境
3. **你最关注什么方向？** — 对齐 Claude 的思考优先级
4. **你每天用 Claude 做什么？** — 决定 Mentor Mode 的权重分配
5. **希望 Claude 是什么角色？** — 导师 / 战友 / 执行者 / 混搭
6. **你最受不了 Claude 什么毛病？** — 动态生成「不要做的」规则
7. **你有什么反复踩的坑？** — 生成特殊触发提醒
8. **你写内容有什么偏好？** — 定制输出风格
9. **常用路径有哪些？** — 让 Claude 记住你的文件系统
10. **还有什么想说的？** — 自由补充

回答完后，`~/CLAUDE.md` 自动生成。随时重新运行覆盖。

## 三层架构详解

### 第一层：灵魂设定

定义 Claude 在你面前的核心人格特质 — 不是说一句「做我的导师」就完了，而是六条行为准则：

- **理性优先**：逻辑和结构，不输出情绪化判断
- **不迎合**：你有问题直接说，不讨好
- **点醒感**：不只说「是什么」，更要说「你没注意到的视角」
- **不评价，启发**：不说「做得好/不好」，说「有没有想过另一种可能」

这些不是随便写的 — 每一条都对应一个真实的 Claude Code 默认行为缺陷。

### 第二层：Mentor Mode

这是最核心的设计 — Claude 不是一个固定的姿态，而是根据你的请求类型**动态切换**：

| 请求类型 | 示例 | Claude 的姿态 |
|---------|------|-------------|
| 决策型 | 换工作、选赛道、重大投入 | 先挑战，再出方案 |
| 执行型 | 写文案、跑脚本、做调研 | 默认直接交付 |
| 信息型 | 查资料、问知识 | 快速给出，不延展 |
| 紧急模式 | 「不要问了直接做」 | 零质疑，百分百执行 |

**三类强制 challenge 场景**：情绪下判断、重复踩坑、明确要复盘 — 不管语气，必须挑战。

### 第三层：记忆系统

Claude Code 原生记忆有两个硬伤：7 天过期、长对话被上下文窗口挤掉。

这套系统用文件持久化绕过限制：

```
~/.claude/pending_review.md  ← 候选记忆，每次会话结束写
                              ← 每次新会话开始审查，确认后写入长期 Memory
```

**写什么**：方法论、决策原则、行为模式 — 三个月后还重要的东西。
**不写什么**：项目进展、一次性信息、临时策略。

## 常见问题

**Q: 这会让 Claude 变得很烦人吗？**

不会。Mentor Mode 的核心是「场景判断优先」— 执行型任务直接交付不废话，信息型不多嘴，只有决策型和三类触发场景才会出手。不是整天怼你。

**Q: 和 Claude Code 自带的 Custom Instructions 有什么区别？**

自带的 Custom Instructions 是一个固定提示，丢了就没了。这套系统：第一，它是一个**状态机**动态切换；第二，记忆系统保证跨会话可累积；第三，交互式向导让你不需要手动调 prompt。

**Q: 我不懂技术，能用吗？**

能。两条路径：要么直接复制模板改 6 个地方，要么运行交互式脚本回答 10 个问题。都不需要写一行代码。

**Q: 用了之后不适合我怎么办？**

删掉 `~/CLAUDE.md` 或清空内容，重开 Claude Code 就恢复。做之前会自动备份旧配置。

**Q: 怎么确认生效了？**

重新打开 Claude Code，打一句「你觉得我应该换工作吗？」— 如果 Claude 没直接说「我觉得可以」，而是先问了你三个问题，就生效了。
